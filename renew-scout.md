# Renew Career スカウト一括送信スキル

Renew Career（renew-career.com）の保存した検索条件を使って、学生候補者に対してスカウトメッセージを一括送信する。完了後にSlackへ結果を通知する。

## 対象アカウント
ユーザーの指示に応じて対象アカウントを判別する:

| アカウント | 判別方法 | Slack Webhook 環境変数 |
|----------|---------|---------------|
| **GARO** | 右上に「株式会社GARO」と表示 | `$SLACK_WEBHOOK_GARO` |
| **Ship** | 右上に「株式会社ship」と表示 | `$SLACK_WEBHOOK_SHIP` |

- 「Shipのスカウト送って」→ Shipアカウントで実行、Ship用Slackに通知
- 「GAROのスカウト送って」または指定なし → GAROアカウントで実行、GARO用Slackに通知
- ページ表示後にスクリーンショットで右上のアカウント名を確認し、指定と異なる場合はユーザーにアカウント切り替えを依頼する

## 前提条件
- Chrome で Renew Career にログイン済みであること（https://renew-career.com/c/dashboard）
- Chrome MCP ツールが利用可能であること

## 実行フロー

### Phase 1: ブラウザ準備
1. `tabs_context_mcp` でタブ状況を確認
2. `tabs_create_mcp` で新しいタブを作成
3. `navigate` で https://renew-career.com/c/scouts に移動
4. ページ読み込み待機（2秒）

### Phase 2: 保存条件で絞り込み
1. `find` で「保存した条件で絞り込む」リンクを見つけてクリック
2. モーダルが開いたら「学生スカウト条件」をクリック
3. `find` で「絞り込む」ボタンを見つけてクリック
4. ページ読み込み待機（2秒）
5. スクリーンショットで候補者一覧が表示されたことを確認

### Phase 3: 候補者を上から順に処理（最大45名）

送信済みリストを `sent_list = []` として管理する。
スカウト可能数の残りが0になったら即座にループを終了する。

各候補者に対して以下を実行:

#### Step 1: 候補者ページを開く
- 候補者一覧から上から順にクリック（リンクを `find` で取得）
- 候補者詳細ページに遷移するまで待機（2秒）

#### Step 2: 興味のある職種を確認
- 左側のプロフィール部分を下にスクロール
- 「興味関心」セクションの「興味のある職種」を `read_page` または `find` で読み取る
- 「エンジニア」が含まれているかどうかを判定

#### Step 3: 求人を選択
- 右側の「求人を選択してください」ドロップダウンを `find` でクリック
- **エンジニアが含まれる場合**: 上から2番目の求人（学生エンジニア募集）を選択
- **エンジニアが含まれない場合**: 上から1番目の求人（事業立ち上げメンバー募集）を選択

#### Step 4: テンプレートを選択
- 右側の「テンプレート」ボタンを `find` でクリック
- テンプレートモーダルが開く
- **エンジニア求人を選択した場合**: 「[スカウト]エンジニア(普通版)」をクリック
- **事業立ち上げメンバー募集を選択した場合**: 「[スカウト]オープンポジション(普通版)」をクリック
- テンプレート内容がプレビューに表示されたことを確認
- 「使用する」ボタンをクリック

#### Step 5: スカウト送信
1. 右側の「スカウトを送信する」ボタンをクリック（スクロールが必要な場合あり）
2. 確認ダイアログ「この内容でスカウトを送信しますか？」が表示される
3. 確認ダイアログ内の求人名とテンプレート内容を確認:
   - 求人とスカウトの職種が一致していることを確認
4. ダイアログ内の「スカウトを送信する」ボタンをクリック
5. 「スカウト送信が完了しました！」が表示されたことを確認

#### Step 6: 結果を記録
- 以下の情報を `sent_list` に追加:
  - 大学名
  - 学部・学科名
  - 選択した求人（事業立ち上げ or エンジニア）
  - 選択したテンプレート（オープンポジション or エンジニア）

#### Step 7: 次の候補者へ
- 完了ダイアログの×ボタンを閉じるか、「スカウト一覧へ戻る」をクリック
- ブラウザの「戻る」で候補者一覧に戻る
- 次の候補者（まだスカウトを送っていない候補者）をクリック
- **注意**: 既にスカウト済みの候補者は「スカウト済み」等の表示になるのでスキップする
- ページ内の候補者を全て処理したら次ページへ（ページネーション）

### Phase 4: Slack通知
全候補者の処理完了後、以下の形式でSlackに通知:

```
📋 Renew Career スカウト送信完了

🕐 実行日時: YYYY/MM/DD HH:MM

【事業立ち上げメンバー募集】N名
- 〇〇大学 △△学部（オープンポジション）
- □□大学 ◇◇学部（オープンポジション）

【学生エンジニア募集】N名
- ◎◎大学 ★★学部（エンジニア）

合計: X名にスカウトを送信しました
残りスカウト数: Y/50
```

Slack通知は Bash ツールで curl を使って、対象アカウントに応じた Webhook URL（環境変数）にPOSTする:
```bash
# GAROの場合
curl -s -X POST -H 'Content-Type: application/json' \
  -d '{"text": "上記メッセージ"}' \
  "$SLACK_WEBHOOK_GARO"

# Shipの場合
curl -s -X POST -H 'Content-Type: application/json' \
  -d '{"text": "上記メッセージ"}' \
  "$SLACK_WEBHOOK_SHIP"
```

## エラーハンドリング
- 候補者の処理でエラーが発生した場合はスキップしてログに残し、次の候補者へ進む
- スカウト可能数が0になったらループを終了し、その旨をSlack通知に含める
- ページ遷移に失敗した場合は再試行（最大2回）
- 各候補者の処理後にスクショで成功確認を行う（3-5人に1回）

## 注意事項
- 候補者が大量の場合、ページネーションに対応する（ページ下部の次ページリンク）
- 1ページあたりの候補者数を確認し、全ページを処理する
- スカウト残数（49/50等）を常に監視し、上限に達したら停止する
- 送信速度は自然なペースで行う（急ぎすぎない）
