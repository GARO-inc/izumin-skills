# Renew Career 応募者スクリーニング＆面談お誘いスキル

Renew Career（renew-career.com）の応募者管理ページで、採用報告が「未対応」の応募者をスクリーニングし、条件を満たす候補者にカジュアル面談お誘いメッセージを送信する。条件を満たさない候補者は「不採用/見送り」に変更する。完了後にSlackへ結果を通知する。

## 対象アカウント
ユーザーの指示に応じて対象アカウントを判別する:

| アカウント | 判別方法 | Slack Webhook 環境変数 |
|----------|---------|---------------|
| **GARO** | 右上に「株式会社GARO」と表示 | `$SLACK_WEBHOOK_GARO` |
| **Ship** | 右上に「株式会社ship」と表示 | `$SLACK_WEBHOOK_SHIP` |

- 「Shipの応募者チェック」→ Shipアカウントで実行、Ship用Slackに通知
- 「GAROの応募者チェック」または指定なし → GAROアカウントで実行、GARO用Slackに通知
- ページ表示後にスクリーンショットで右上のアカウント名を確認し、指定と異なる場合はユーザーにアカウント切り替えを依頼する

## 前提条件
- Chrome で Renew Career にログイン済みであること
- Chrome MCP ツールが利用可能であること

## スクリーニング条件（すべて満たす必要あり）
1. **都道府県**: 東京都、千葉県、神奈川県のいずれか
2. **学校名**: 英語ではないこと（日本語の学校名であること）
3. **出勤可能日数**: 週2日以上
4. **合計稼働時間**: 週15時間以上
5. **継続可能期間**: 3ヶ月以上

## 実行フロー

### Phase 1: ブラウザ準備
1. `tabs_context_mcp` でタブ状況を確認
2. `tabs_create_mcp` で新しいタブを作成
3. `navigate` で https://renew-career.com/c/applicants に移動
4. ページ読み込み待機（2秒）

### Phase 2: 未対応の応募者を特定
1. スクリーンショットで応募者一覧を確認
2. 採用報告カラムが「未対応」の応募者を上から順に処理
3. 「未対応」が0名ならPhase 4へスキップ

### Phase 3: 各応募者を処理

処理結果リストを管理:
- `invited_list = []`（面談お誘い送信済み）
- `rejected_list = []`（不採用/見送り）

各「未対応」応募者に対して以下を実行:

#### Step 1: 応募者詳細を開く
- 応募者の「詳細」ボタンをクリック
- 詳細ページに遷移するまで待機（2秒）

#### Step 2: プロフィール確認・条件チェック
- 基本情報セクションから以下を読み取る:
  - **都道府県**: 東京都/千葉県/神奈川県のいずれかか？
  - **学校名**: 英語名（アルファベットのみ）ではないか？
- 下にスクロールしてインターン条件セクションから以下を読み取る:
  - **出勤可能日数**: 週2日以上か？
  - **合計稼働時間**: 週15時間以上か？
  - **継続可能期間**: 3ヶ月以上か？

#### Step 3A: 条件を満たす場合 → 面談お誘いメッセージ送信
1. 右上の「メッセージに移動する」をクリック（またはURLで `https://renew-career.com/c/messages/{応募者ID}` に直接遷移）
2. メッセージページに遷移するまで待機（2秒）
3. 下側の「テンプレート」ボタンをクリック
4. テンプレートモーダルで「①面談調整(大池)」をクリック
5. 「使用する」ボタンをクリック
6. メッセージ内容がプレビューに表示されたことを確認
7. 「メッセージ送信」ボタンをクリック
8. 送信完了を確認
9. 応募者詳細ページに戻る（`https://renew-career.com/c/applicants/{応募者ID}/profile`）
10. ステータスを「未対応」→「連絡中」に変更（下記「ステータス変更方法」参照）
11. `invited_list` に記録（名前、大学名、応募求人）

#### Step 3B: 条件を満たさない場合 → 不採用/見送り
1. ステータスを「未対応」→「不採用/見送り」に変更（下記「ステータス変更方法」参照）
2. `rejected_list` に記録（名前、大学名、不合格理由）

#### ステータス変更方法（重要）
Renew Career のステータスドロップダウンはカスタムUIのため、通常のクリック操作では変更できない。
`javascript_tool` を使って以下のJSコードでフォーム送信する:

```javascript
const select = document.querySelector('select[name="application_status"]');
const form = select.closest('form');
select.value = '変更先の値';  // 2:連絡中, 6:不採用/見送り
if (form) { form.submit(); }
```

ステータス値一覧:
- `1`: 未対応
- `2`: 連絡中
- `3`: 書類選考
- `4`: 面接
- `6`: 不採用/見送り
- `10`: 選考辞退/連絡なし

#### Step 4: 次の応募者へ
- 応募者管理ページ（/c/applicants）に戻る
- 次の「未対応」応募者を処理
- 全「未対応」応募者を処理し終えるまで繰り返す

### Phase 4: Slack通知
全応募者の処理完了後、以下の形式でSlackに通知:

```
📋 Renew Career 応募者スクリーニング完了

🕐 実行日時: YYYY/MM/DD HH:MM

【面談お誘い送信】N名
- 〇〇さん（△△大学 / 求人名）
- □□さん（◇◇大学 / 求人名）

【不採用/見送り】N名
- ◎◎さん（★★大学 / 理由: 学校名が英語）
- ●●さん（■■大学 / 理由: 稼働時間不足）

合計: X名処理（面談お誘い: Y名 / 見送り: Z名）
```

Slack通知は Bash ツールで curl を使って、対象アカウントに応じた Webhook URL（環境変数）にPOSTする:
```bash
# GAROの場合
curl -s -X POST -H 'Content-Type: application/json' \
  -d '{"text": "上記メッセージ"}' \
  "$SLACK_WEBHOOK_GARO"

# Shipの場合
curl -s -X POST -H 'Content-Type: application/json' \
  -d '{"text": "上記メッセージ"}' \
  "$SLACK_WEBHOOK_SHIP"
```

## エラーハンドリング
- 応募者の処理でエラーが発生した場合はスキップしてログに残し、次の応募者へ進む
- ページ遷移に失敗した場合は再試行（最大2回）
- 各応募者の処理後にスクショで成功確認を行う

## 注意事項
- 「未対応」以外のステータスの応募者はスキップする
- 条件判定は厳密に行い、不明な場合はスクリーンショットで確認する
- 学校名の英語判定: アルファベット・スペースのみで構成されている場合は英語とみなす
- 出勤可能日数は「週N日」の数値で判定（「週2日〜OK」なら2日以上で合格）
- 継続可能期間は「N ヶ月以上」の数値で判定
